# 数据源加权与冲突裁决

## 默认来源分工

### 官方说明 / 游戏内截图

用于确认当前版本规则、热修、技能文本、天赋文本、物品文本、地下城手册和 Boss 机制。若用户提供游戏内截图，优先级很高，但仍要注意语言版本、难度和是否是 PTR/正式服。

### Wowhead

用于确认职业指南、天赋、技能/物品页面、掉落、套装、版本改动和基础理论。适合建立“通用起点”，但同站不同页面可能更新时间不同，必须比较日期。

### Icy Veins / Method / 职业社区资源

用于补充循环解释、开场、天赋分歧和实战建议。适合作为导师式解释的材料，但不要在与日志/模拟冲突时无条件照搬。

### Archon / Warcraft Logs

用于确认高分玩家、团本 Boss、M+ 统计、天赋热度、饰品热度和实际打法趋势。结论属于“实战样本层”，可能受队伍配置、击杀时长、层数、路线、词缀、喂资源影响。

### Mythicstats

用于观察 M+ 高分角色的装备、天赋和构筑趋势。适合回答“高层常见怎么带”，不适合直接回答“小号现在应该怎么带”。

### SimulationCraft / Raidbots

用于个人角色装备、饰品、属性、天赋精确比较。涉及个人收益排序时，它比通用攻略更重要。

### Bloodmallet

用于饰品、消耗品、副属性等理论模拟参考。通常不是个人角色结论，也不一定代表 M+ 或特定 Boss。

## 按问题加权

- `入门循环`：Wowhead / Icy Veins / Method 优先，再用日志确认主流构筑。
- `为什么这么打`：技能/天赋文本 + 指南解释 + 日志趋势共同推导。
- `M+ 打法`：Archon / Warcraft Logs M+ / Mythicstats 优先，攻略作为机制解释。
- `团本 Boss`：Warcraft Logs 对应 Boss + 攻略 Boss 建议优先。
- `装备饰品`：先看个人 `/simc`，没有时用 Wowhead / Bloodmallet / 日志热度给方向。
- `当前天赋是否合理`：先看用户天赋，再对照当前主流构筑和玩家目标。

## 冲突裁决

出现冲突时按顺序处理：

1. 确认资料是否属于当前正式服版本，不要混用 PTR、旧赛季或旧补丁。
2. 比较页面更新时间；同站页面冲突时优先采用更新时间更近、语境更匹配的页面。
3. 确认场景：单体、AOE、顺劈、M+、团本、Boss、目标数、层数、怪包寿命。
4. 确认玩家阶段：小号、普通层、冲层、开荒、farm、双修。
5. 对装备/饰品冲突，优先要求个人 `/simc`。
6. 仍不确定时，给条件化结论，不要假装唯一正确。

## 必须提示样本偏差的情况

- 数据来自极高层 M+ 或极高解析日志。
- 数据来自单 Boss，而用户问通用团本或 M+。
- 玩家装等、套装、饰品、职业熟练度明显低于样本。
- 结论依赖特定队伍拉法、战斗时长、外部增益或团本战术。
- Bloodmallet/SimC 模型与实际战斗环境不同。

## 引用输出

联网后最终回答应简短列出来源类型或链接。不要大段复制攻略原文；用自己的话解释机制。
