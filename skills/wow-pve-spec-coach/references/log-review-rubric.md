# Warcraft Logs 复盘检查项

## 通用顺序

1. 战斗是否有效：死亡、掉线、任务分配、战斗时长、团队策略是否异常。
2. 天赋和装备是否符合问题场景：不要拿 AOE 天赋复盘纯单体，或反过来。
3. 核心覆盖率：DoT、HoT、buff、debuff、护盾、减伤、宠物/召唤物 uptime。
4. 爆发次数：按战斗时长理论可用几次，实际用了几次，是否错过关键窗口。
5. 资源浪费：能量/怒气/符文/法力/豆/魂片/圣能/充能是否溢出。
6. 技能优先级：关键技能是否被延迟，填充技能是否挤占核心窗口。
7. 目标选择：是否打错主目标、未利用顺劈、过度铺垫快死目标。
8. 生存与功能性：死亡原因、防御技能、打断、驱散、控制、免疫、位移。

## DPS 重点

- 核心 DoT/debuff/buff 覆盖率。
- 主要冷却的使用次数和对齐情况。
- 大技能是否打在增伤、饰品、药水、斩杀或大怪包窗口。
- AOE 职业要看目标数和怪包寿命，不能只看总伤。
- 单体职业要看是否因跑位、转火或机制丢失爆发窗口。

## 治疗重点

- 死亡是否可治疗，还是玩家站错/没开减伤。
- 大技能是否提前规划，而不是死人后才交。
- 法力曲线是否健康。
- HoT/护盾/减伤覆盖是否服务于真实伤害事件。
- 是否过度刷无效治疗，牺牲驱散、打断、走位或伤害。

## 坦克重点

- 主动减伤覆盖与危险尖刺是否对齐。
- 大减伤是否留给真正危险波次。
- 风筝、位移、拉怪节奏是否导致队友输出或治疗压力异常。
- 死亡前 10-15 秒是否有资源、防御、外部减伤可用。

## 输出方式

不要只说“覆盖率低”。要写：

```text
现象：某覆盖率/次数偏低
机制后果：它影响了什么资源、增伤或生存链
可能原因：忘按、刷新时机错、被机制打断、天赋不匹配、轴没规划
训练目标：下一把只盯一个可执行动作
```
