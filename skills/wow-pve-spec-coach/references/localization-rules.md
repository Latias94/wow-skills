# 中文术语与本地化规则

## 目标

让中文客户端玩家能直接读懂攻略。英文资料可以作为来源，但最终答案必须转换成中文玩家熟悉的表达。

## 默认规则

- 用户中文提问：中文主导，英文只做第一次括号对照。
- 用户英文提问：英文主导，可不翻译。
- 用户中英混合：按用户主要语言输出；如果用户明显在看英文攻略，可以给中英对照。

## 术语格式

首次出现：

```text
中文名（English original）
```

后文：

```text
中文名
```

中文名不确定：

```text
中文描述（English original，中文名待核）
```

不要使用：

```text
English original（中文名）
```

除非用户明确要求英文为主。

## 缩写解释

第一次出现时解释：

- `AOE（群体伤害）`
- `ST（单体）`
- `Cleave（顺劈/少目标顺劈）`
- `DoT（持续伤害）`
- `HoT（持续治疗）`
- `GCD（公共冷却）`
- `CD（冷却时间）`
- `WCL（Warcraft Logs 战斗日志）`
- `SimC（SimulationCraft 模拟）`
- `WA（WeakAuras 监控）`

后文可以使用缩写。

## 英文资料转写

处理 Wowhead、Icy Veins、Method、Archon、WCL、Raidbots 等英文资料时：

1. 先理解机制，不要逐字翻译。
2. 把技能/天赋/装备名转换为中文优先格式。
3. 把英文优先级翻译成中文动作。
4. 保留英文名只用于对照日志、插件、英文攻略和天赋导入。

## 中文名核对来源

优先级：

1. 用户提供的游戏内截图、天赋截图、装备截图。
2. 官方中文客户端文本或官方中文网页。
3. Wowhead 中文页面或可切换语言页面。
4. 常见中文社区通用译名。
5. 无法确认时使用“中文描述（英文原名，中文名待核）”。

## 示例：刺杀贼入门表达

推荐：

```text
先保持 锁喉（Garrote）和 割裂（Rupture）。
连击点到 5+ 后打 毒伤（Envenom）。
群体场景先确认主目标流血，再根据当前天赋使用 猩红风暴（Crimson Tempest）或其他 AOE 铺伤害方式。
死亡印记（Deathmark）和 君王之灾（Kingsbane）要交给活得够久的目标。
```

不推荐：

```text
Maintain Garrote/Rupture, use Envenom at 5+ CP, sync Deathmark with Kingsbane.
```

## 输出自检

交付前快速检查：

- 中文玩家是否能不查词就理解核心循环？
- 是否有连续 2 个以上未解释英文技能/天赋名？
- 是否把英文资料里的术语直接搬进了正文？
- 是否给了必要的中英对照，方便用户对照 WA/WCL/英文攻略？
