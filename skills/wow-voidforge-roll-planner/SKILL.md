# WoW Voidforge Roll Planner

## 核心边界

- 这是一个 `roll 币 / 晦暗虚空核心 / 宏伟宝库代币` 决策 skill，不是通用职业攻略 skill。
- 默认优先回答：`这周拿什么`、`这周把币投哪里`、`要不要继续追同池`、`要不要拿 6 代币换额外核心`。
- 允许把 `官方英雄榜 / 角色页 URL` 当作角色当前状态输入源；它是 `/simc` 之后的次优选择，但不能替代宝库候选、背包待选装备和 R 币去重历史。
- `BIS`、属性、Hero talents、套装、制作装只作为决策证据；除非用户明确要求，不要把答案写成长篇专精综述。
- 如果用户给的是二选一 / 三选一选择题，第一屏先给结论与动作顺序，再解释理由。
- 如果用户问的是 `roll 规划`，不要把主要篇幅花在纯装备强弱点评上；要把重点放在 `活动优先级`、`同池集中`、`可达性`、`1 币 vs 2 币`、`宝库代币`。
- 如果用户没有给出明确选择题，只给了角色状态、玩法目标或 `/simc`，默认输出一份具体 `1-3 周 R 币规划`，不要只评价当前装备。
- 所有宝库建议都必须放进长期 R 币路线里判断：若宝库装备会和最高优先级 roll 池的核心目标撞槽，要显式评估重复槽位风险。
- 对同一个副本 / Boss，必须考虑 `拾取专精` 是否会改变 bonus roll 候选池；若换拾取专精能保留目标且减少垃圾池，要纳入建议。
- 默认输出应是 `可执行计划`，而不是只有结论：至少覆盖 `当前差距`、`本周核心预算`、`未来 1-3 周路线`、`和高端样本/攻略的差距`。
- 制作装不是默认前置动作；若问题涉及制造业，先按 `当前战力优先 / 长期保值优先 / 平衡` 判断是 `现在做`、`等本周结果后再做`、还是 `暂时不做`。

## 问题类型优先级

先判断用户是哪一类问题，再决定展开深度：

1. `本周选择题`
- 例如：宝库两件选哪件、拿装备还是拿 6 代币、这周额外核心投哪个活动、要不要继续追同一个池子
- 这类问题必须先直接给结论，再补理由
- 输出里至少一半篇幅应放在 `本周动作` 和 `roll 币规划`

2. `1-3 周路线题`
- 例如：未来几周应该先压哪个副本 / 哪个 Boss / 哪个难度
- 这类问题可以展开评分逻辑，但仍应围绕 `roll 投向与连续追击`

3. `精细比较题`
- 例如：两把武器、两个饰品、宝库装备 vs 6 代币换额外核心
- 这类问题若接近，优先建议 `/simc`
- 没有 `/simc` 时可以给方向，但不要假装能精确到小数点

4. `背景解释题`
- 只解释与当前 `roll 决策` 直接相关的专精逻辑
- 除非用户明确要求，不要默认扩写成完整 BIS / 属性 / 天赋攻略

## 快速流程

0. 先判断输出语言。
- 若用户显式要求某种语言，严格遵循该语言
- 若用户没有显式要求，默认跟随用户输入的主语言
- `中文输出`：副本名、Boss 名、装备名默认优先用中文，必要时第一次出现可附英文
- `英文输出`：副本名、Boss 名、装备名默认优先用英文，必要时第一次出现可附中文
- 如果用户输入混合中英且没有明确要求，默认采用问题主体语言；若仍然不清晰，可在首段写明语言假设
- 不要在没有必要时中英来回切换；保持整篇答案语言一致

0.5 先识别问题类型并复述当前决策题。
- 当前是 `本周选择题`、`1-3 周路线题`、`精细比较题`、还是 `背景解释题`
- 在开头先复述用户真正要做的动作：`拿哪件`、`换不换 6 代币`、`本周 roll 投哪里`、`要不要继续追同池`
- 若用户的问题本质是周决策，不要把回答写成完整配装攻略
- 若用户没有具体问题，默认把问题改写成：`未来 1-3 周 R 币应该集中投哪里，本周宝库应如何配合这条路线`

1. 先建立玩家画像。
优先收集这些信息；缺失时可先按保守假设继续，但必须在答案里写明假设：
- 职业专精与 Hero talents
- 主号还是小号
- 主要内容定位：`团本` / `M+` / `双修`
- 当前装等
- 当前 2 件套 / 4 件套进度
- 当前双饰品、武器、关键稀有装
- 本周可稳定通关的团本难度
- 本周可稳定完成的 M+ 层数
- 当前宏伟宝库候选
- 当前目标偏好：`当前战力优先` / `长期保值优先` / `平衡`
- 当前是否大致有多余火花，以及是否有可用纹章 / 预算支持制作；不需要精确数量
- 当前已做制作装；若有 `/simc`，优先直接从导出读取
- 若用户提供 `官方英雄榜 / 角色页 URL`，记录它来自哪个地区站点；优先接受 Blizzard 官方角色页，而不是第三方数据库页
- 当前拾取专精，以及玩家是否愿意在 roll 前切换拾取专精
- 当前与高端样本 / 攻略推荐相比，最缺的是武器、饰品、套装、off-piece 还是属性分布
- 过去几周各活动 / 各难度已经通过晦暗虚空核心拿到过哪些物品
- 若用户提供 `/simc`，记录它是 `直接粘贴`、`附件文本文件`、还是 `本地路径文件`

一律按输入完整度分三档处理：
- `基础档`：只有专精、Hero talents、主玩法、装等、可打内容。只给方向性路线，不给过细“这两件谁强 0.3%”结论
- `进阶档`：再加当前双饰品、武器、套装进度、最弱槽位、宝库候选，或提供官方英雄榜 URL。可以给较稳的周计划与宝库判断
- `精确档`：提供 `/simc` 导出、宝库截图、当前待选装备。可以做武器 / 饰品 / 戒指 / 制作装之间的精细比较

若用户要补 `/simc`，默认按这个优先级接收，不要求固定文件名：
- `第一优先`：直接把 `/simc` 文本粘贴到消息里，最好用代码块包起来
- `第二优先`：提供 `官方英雄榜 / 角色页 URL`，用于快速读取当前装备、专精、装等与进度
- `第三优先`：提供一个纯文本附件文件，例如 `.txt` / `.simc`
- `第四优先`：给出本地文件路径，让 agent 读取
- 不要要求用户必须把 `/simc` 放到某个固定目录；目录规范只能作为可选建议，不能作为硬门槛

2. 先核对版本。
确认补丁、赛季副本池、团本、催化剂规则、晦暗虚空核心获取上限与消耗。用户给了来源时，优先读取用户指定来源。

3. 先锁定当前要解决的 `roll / 宝库 / 代币` 决策。
- 本周宝库是 `直接拿装备` 还是 `拿 6 代币换额外核心`
- 本周额外核心该投 `哪个副本 / 哪个 Boss / 哪个难度`
- 本周投入该活动时应该使用 `哪个拾取专精`
- 是否应该 `继续追同一个活动 / 难度的池子`
- 是否因为 `制作装 / 催化剂 / 已有去重进度` 改变当前投向
- 若存在一个明显 S 级长期 roll 池，先判断宝库候选是否会和该池里的核心目标撞槽；撞槽时不要只看本周装等提升
- 把本周核心预算写清楚：默认按 `每周 2 个核心` 规划；若建议拿 `6 代币`，要明确本周是否会变成 `3 个核心`
- 若涉及制造业，不要默认把制作装放在本周动作最前面；先判断玩家目标偏好，再给出 `现在做 / 等本周结果后做 / 暂时不做`

4. 再按决策需要核对专精，而不是先写一篇职业综述。
- 只读取会影响当前决策的内容：`BiS Gear`、`Stats`、`Talent Builds`、`Hero Talents`
- 优先回答：哪些高价值目标真正影响 `roll 优先级`
- 若 Hero talents 不改变装备池，要明确写“装备池基本不变”，不要把篇幅浪费在重复分支上
- 若页面结论冲突，以“页面更新时间 + 当前实战流行度 + 物品机制强度 + 可获取性”综合判断

5. 明确 Hero talents 与内容场景。
至少区分：
- 团本主流 Hero talents
- M+ 主流 Hero talents
- 若两套 Hero talents 的装备池相同，要明确写“装备池基本不变，只是属性权重或内容偏好有轻微变化”
- 若两套 Hero talents 的属性权重明显不同，必须拆成两套 roll 建议

6. 先判断该看哪类实战样本。
- `团本玩家`：优先看 Wowhead 团本向 BiS / 属性逻辑，再用 WCL 的团本日志、Boss 排名、Boss 维度构筑去验证
- `M+ 玩家`：优先看 Wowhead M+ 向建议，再用 Mythicstats 与 WCL Mythic+ 统计页交叉验证高分钥石样本
- `双修玩家`：先分别看团本与 M+ 主流，再找重叠装备池；优先追“两个场景都强”的目标
- 若一个物品只在极端速推、特定 Boss、特定拉法中强，不要直接当作通用结论

7. 先排除不该 roll 的槽位。
默认降权这些槽位：
- 已有明确可制造毕业或准毕业方案的槽位；但不要因此自动推荐立刻做制作装
- 高概率要用催化剂转套装的槽位
- 只是普通绿字升级、没有特效或稀缺性的槽位
- 用户当前根本打不到对应难度的 Boss 或钥石层数

8. 再识别高价值目标。
通常按这个顺序看价值：
- 特效饰品 / 稀缺饰品
- 稀缺武器 / 稀缺副手
- 不可转套的高质量散件
- 高质量戒指 / 项链
- 普通护甲散件

9. 对高价值活动做拾取专精池检查。
- 对玩家职业的每个可用专精，枚举该活动 / 难度 / Boss 的可 roll 掉落池
- 确认目标物品是否仍在该拾取专精池内
- 计算或半定量比较 `目标数量 / 剩余可 roll 池大小`
- 若某个拾取专精会排除其他 S 级目标，不能只为了缩池而推荐
- 若来源页面无法可靠确认，要标注不确定，并建议用户用游戏内冒险指南、Wowhead 掉落过滤或相关 addon 截图复核

10. 把玩家分层后再给建议。
至少区分三档：
- `M+ 为主`：优先推荐 1 币的大秘境目标；除非 2 币团本目标明显超模且可稳定击杀
- `英雄团稳定`：可把可稳定 farm 的英雄 Boss 纳入主计划
- `史诗团稳定`：才把史诗尾王或史诗后段 Boss 作为主计划；否则只能当远期目标，不要当默认建议

11. 最后再处理宝库与周内决策顺序。
默认顺序：
- 先看清楚本周宝库所有候选
- 先确定未来 1-3 周的主 roll 池和主目标槽位
- 若宝库给出的是不在主 roll 池里、且很难通过稳定 farm 获得的直接 BIS / 稀缺来源 BIS，优先拿宝库装备
- 若宝库给出的装备和主 roll 池核心目标撞槽，必须评估 `拿了以后继续追该池会不会大幅增加重复浪费`
- 若宝库没有明确升级，而你仍有高价值 roll 目标，倾向拿 6 代币换额外晦暗虚空核心
- 若两者接近或牵涉 trinket / 武器 / 属性联动，建议用户导出 `/simc` 再比较
- 若要靠宝库代币换核心来补本周额外 roll，则必须先做出“拿代币”的选择；但在此之前，可以先完成对本周宝库候选的全部评估
- 若用户只给了笼统装备情况，不要装作能得出精确 upgrade 顺序；要把结论表述成“方向性建议”
- 若不需要本周立刻领取宝库，建议先完成本周计划中的关键 roll，再决定是否领取宝库装备；但如果要用 6 代币换本周额外核心，则必须先选择代币

## 判定规则

用一个简单的优先级框架给每个活动打分，不要只看理论 BIS：

`综合优先级 = 物品强度 x 可达性 x 槽位稀缺度 x 同副本多目标收益 x 当前角色阶段修正 / roll 成本`

在输出时明确解释这些修正项：
- `套装可转化惩罚`：头 / 肩 / 胸 / 手 / 腿都可能被催化剂覆盖，除非该专精明确把其中某个槽位当固定散件，否则不要优先用 roll 币追
- `可制造惩罚`：制作装能解决的槽位，默认降低 roll 币优先级；但这不等于“现在就该做制作装”，还要看玩家目标、宝库与主池撞槽风险
- `当前装等修正`：低装等 / 小号更需要尽快补齐关键弱槽与 4 件套路径，因此会提高“1 币、可重复 farm、多个散件可用”的 M+ 目标权重
- `主号冲榜修正`：高装等主号在已有基本成型后，会显著提高稀缺团本饰品、武器、尾王掉落的权重
- `内容定位修正`：团本玩家提高团本 Boss 特定强装权重；M+ 玩家提高 1 币高复刷副本权重；双修玩家优先追两个场景重叠的核心目标
- `目标偏好修正`：`当前战力优先` 可接受一定制作浪费换取本周强度；`长期保值优先` 更看重避免火花 / 纹章砸进短期可能被宝库或 roll 顶掉的槽位；未说明时默认按 `平衡偏保守`
- `可达性修正`：尾王、史诗后段 Boss、需要固定团复杀的目标，显著降权
- `同副本多目标加成`：一个副本里若同时有目标饰品 + 目标戒指 / 散件，该副本优先级会抬高
- `低保协同加成`：若宏伟宝库开不出想要装备，可用 6 代币换 1 个额外晦暗虚空核心；当剩余目标仍是高价值物品时，这个额外 roll 通常优于低价值货币奖励
- `难度分层修正`：晦暗虚空核心的去重是按难度维度生效；建议优先把 roll 集中在你最常刷、最想要对应掉落轨道的难度上，不要过早拆散到多个难度池
- `连续追击加成`：若某个副本 / Boss 的同难度掉落池里，你已经通过晦暗虚空核心拿到了若干非目标物品，那么后续继续在同活动、同难度 roll 时，命中剩余目标的概率会上升；因此对单一高价值目标，往往应该连续数周集中投入，而不是每周改追不同活动
- `去重范围修正`：默认只把“通过晦暗虚空核心拿到的物品”视为从该 bonus roll 池中移除；不要默认把自然掉落、宏伟宝库、队友交易得到的同名物品也当成已被去重，除非官方后续明确说明
- `长期池锚定修正`：如果一个副本 / Boss 同时包含多个长期强目标，且玩家能稳定 farm，它可以成为未来 1-3 周的主 roll 池；本周宝库选择必须服从这个主池规划
- `重复槽位风险修正`：如果宝库装备和主 roll 池里最想要的目标是同一槽位，拿宝库可能降低后续继续追该池的边际收益；不能只按本周装等提升判断
- `行动顺序修正`：默认先制定主 roll 池，再评估宝库；如果不急着用 6 代币换核心，可以先做关键 roll 再领宝库，避免先拿了和 roll 结果冲突的装备
- `拾取专精池修正`：同一活动在不同拾取专精下可 roll 池可能不同；若某个拾取专精保留目标且减少非目标装备，它会提高该活动本周期望
- `拾取专精排除风险`：如果切换拾取专精会把另一个同池 S 级目标排除，或引入玩家不想要的武器 / 饰品类型，必须降权或拆成两套建议
- `英雄天赋修正`：若 Hero talents 只影响玩法、不影响装备池，不要人为拆出两套完全不同的 BIS；若 Hero talents 改变关键副属性需求、关键 trinket 强度或武器选择，再单独拆分
- `日志偏差修正`：WCL 的团本与 M+ 页面都可能被高解析、速推、特定 Boss、特定路线或特定拉法放大；Mythicstats 的高分钥石样本也会偏向高端路线。它们都不能脱离可达性直接照抄
- `宝库替代修正`：如果宝库已经给出你原本计划用核心追的高价值装备，则该活动本周 roll 优先级直接下降
- `现实毕业修正`：若某理论 BIS 只来自 `2 币 + 单目标 + 低可达 + 无去重进展` 的池子，它更适合当远期目标，而不是当前主路线；这时要主动评估现实毕业与制造替代
- `制造时机修正`：制造建议必须明确落到 `现在做`、`等本周关键 roll / 宝库后再做`、`暂时不做`、或 `只有当前战力优先时才做`
- `模拟必要性修正`：若多个选项都看起来接近，且至少一个涉及 trinket、武器、断点急速、套装拆分、制作装替换，则不要假装能肉眼精确判断，应建议 `/simc`
- `输入完整度修正`：输入越粗，结论越应该保守；没有 `/simc` 时可以定方向，但不要声称能给出严丝合缝的精确 upgrade 排名

## 分析顺序

1. 先复述用户现在真正要做的决策。
至少明确：
- 这是 `拿哪件宝库 / 换不换 6 代币 / 本周 roll 投哪里 / 要不要继续追同池` 中的哪一种
- 这是 `本周选择题` 还是 `1-3 周路线题`
- 如果用户只是在做二选一，不要先写整套 BIS 盘点
- 如果用户没有具体问题，默认输出完整 R 币路线：`本周先做什么、宝库何时领、未来 1-3 周继续追哪里`

2. 先列出当前需要比较的候选项。
例如：
- 宝库候选 A / B / 6 代币
- 大秘境活动 X / 团本 Boss Y
- 继续追当前池子 / 改追新池子
- 主 roll 池里的核心目标是否和宝库候选撞槽
- 对同一活动，用当前拾取专精还是切换拾取专精

3. 先确认版本规则与 bonus roll 机制。
- 当前补丁与赛季池
- 晦暗虚空核心来源、成本、上限
- 6 代币换额外核心的前提
- 同活动 / 同难度去重与连续追击逻辑
- bonus roll 是否按当前拾取专精决定候选池；若版本规则不确定，必须联网或用用户指定来源核实

4. 再按当前决策需要，列出该专精真正 relevant 的高价值目标。
至少覆盖：
- 武器 / 饰品 / 稀缺 off-piece
- 与本周候选直接冲突的槽位
- 不要为了完整而把所有普通散件展开成长表

4.5 对每个主 roll 池做拾取专精路由。
- 列出该职业可选拾取专精
- 对每个拾取专精，列出目标是否在池内、池子是否更小、会不会排除其他高价值目标
- 输出推荐拾取专精：`用 A 专精 roll`、`保持当前专精`、或 `不要为了缩池切专精`

5. 再按 Hero talents 标注差异。
- 团本主流 Hero talents 是什么
- M+ 主流 Hero talents 是什么
- 两套 Hero talents 的属性优先级是否不同
- 两套 Hero talents 的关键装备是否不同
- 若装备池基本不变，要明确写“本题不需要拆两套长表”

6. 先按内容定位选数据源。
- 团本玩家：以 Wowhead + WCL 为主
- M+ 玩家：以 Wowhead + Mythicstats + WCL Mythic+ 为主
- 双修玩家：三者都看，但优先找重叠结论

7. 标注每个槽位是否可被以下方案替代：
- 套装催化剂
- 制作装
- 宏伟宝库直接补位
- 低成本常刷副本

8. 结合角色阶段重新排序。
至少回答：
- 这是主号冲 BIS，还是小号补装等
- 当前最缺的是“关键特效槽位”还是“整体装等”
- 当前是否已经有 4 件套
- 当前是否已经有能长期使用的主副手 / 双饰品
- 当前目标偏好是更偏 `当前战力优先` 还是 `长期保值优先`
- 当前与高端样本 / 攻略推荐的差距主要在哪：武器、饰品、套装位、off-piece、属性分布、拾取专精

9. 找出真正值得吃 roll 币的活动。
不要只写“这个物品 BIS”；要写清楚：
- 来自哪个副本 / Boss
- 该活动是 1 币还是 2 币
- 该活动是否现实可 farm
- 该活动里是否还有第二、第三目标
- 建议用哪个拾取专精 roll，以及为什么
- 它是否适合作为未来 1-3 周连续投入的主池
- 如果用户已经连续几周在同活动、同难度用晦暗虚空核心拿到过其他物品，要把“剩余目标命中率上升”明确写进结论

9.5 若某理论 BIS 明显低可达、低池质量或需要 `2 币` 长期硬追，单独判断它是否仍应当作当前主路线目标。
- 如果不是，明确写出 `现实毕业路线`
- 如果该槽位有合理制作替代，写明是 `现在做`、`等本周结果后再做`、还是 `暂时不做`
- 如果没有 `/simc`，不要假装能精确比较制作装与掉落装的细微差异；保持方向性结论

10. 单独讨论套装位。
若该专精可以把 4 件套稳定放在头 / 肩 / 胸 / 手，那么腿通常是优先保留给高质量散件的候选；若样本数据显示高端玩家常用腿套，则明确写出“该专精腿位不作为默认 off-piece”。

11. 单独处理宝库决策。
至少回答：
- 本周宝库有没有“直接拿就对”的选项
- 如果没有，拿 6 代币换额外核心是否优于其他安慰奖
- 是否应该先做本周额外 roll，再决定后续制作装、纹章升级、催化剂分配
- 宝库候选是否与长期主 roll 池的核心目标撞槽；若撞槽，要说明拿它会不会降低继续追主池的价值
- 若宝库选择牵涉饰品 / 武器 / 多件纯绿字散件比较，优先建议用户补 `/simc` 或至少补当前装备详情

12. 输出最终路线时按阶段写。
至少给出：
- `第一优先`：现在就该 roll 的内容
- `第二优先`：第一优先拿到后再 roll 的内容
- `不建议主动 roll`：低价值、可转套、可制造、难以 farm 的内容
- `本周行动顺序`：先 roll、先领宝库、还是先拿 6 代币换核心
- `本周核心预算`：默认把 `2 个核心` 如何分配写清楚；若建议 6 代币，写清 `3 个核心` 的分配方式
- `未来 1-3 周路线`：主池是否继续、何时转池、触发转池的条件
- `当前 vs 高端样本差距`：至少覆盖武器、饰品、套装 / off-piece、属性 / Hero talents
- `需要 /simc 才能严谨决定`：列出肉眼不该硬判的分歧点
- 若用户提供了往周已通过晦暗虚空核心拿到的物品，要把“为什么继续追同一个本 / 为什么该换本”写成显式说明

## 输出模板

按下面结构输出，避免变成散乱攻略：

### 0. 玩家画像、问题类型与假设
- 主号 / 小号
- 团本 / M+ / 双修
- 当前问题是 `本周选择题`、`1-3 周路线题`、还是 `精细比较题`
- 输入完整度：基础档 / 进阶档 / 精确档
- 目标偏好：`当前战力优先` / `长期保值优先` / `平衡`
- 当前装等与套装进度
- 团本 / M+ 可达性
- 当前宝库与制作装状态
- 是否大致有多余火花、可用纹章 / 预算
- 是否提供官方英雄榜 / 角色页 URL
- 当前武器 / 饰品 / off-piece / 属性分布与高端样本的大致差距
- 若缺信息，写清默认假设

### 1. 一句话结论与本周动作
- 直接回答：这周 `拿哪件 / 换不换 6 代币 / roll 投哪里`
- 如果有先后顺序，写成 `先做什么，再做什么`
- 如果用户只是二选一，这一节必须出现在前面
- 如果用户没有明确选择题，输出 `本周行动顺序 + 未来 1-3 周主 roll 池`
- 对每个建议 roll 的活动，写明推荐拾取专精
- 若能判断，直接写本周 `2 个核心` 或 `3 个核心` 的分配

### 2. 当前状态与高端差距
- 当前长期可用武器 / 饰品 / 套装 / off-piece
- 当前最弱槽位
- 与攻略 / 高端样本相比，哪些关键毕业位还没成型
- 当前属性分布、Hero talents 与主流方案是否偏离

### 3. 规则确认
- 当前补丁与赛季池
- 晦暗虚空核心获取与消耗
- 宝库 6 代币换额外 roll 的适用条件

### 4. 决策依据
- 只写与当前决策直接相关的属性 / 毕业方向
- Hero talents 分流结论
- 数据源加权结论：这是更偏 `Wowhead`、`Mythicstats`、`WCL 团本`、还是 `WCL Mythic+`
- 如果相关，补充 Bloodmallet 的理论饰品 / 属性结论
- 套装位 / off-piece 方案
- 理论 BIS 与现实毕业是否一致
- 制作装是否会挤占某些掉落位
- 宝库候选是否会和长期主 roll 池撞槽
- 如果某些专精背景知识不影响当前决策，明确省略，不要凑成长篇综述

### 5. 本周行动顺序与核心预算
- 本周先做什么，再做什么
- `2 个核心` 默认怎么分配；若建议拿 6 代币，写成 `3 个核心` 的分配
- 建议打几次哪个 M+ 或打哪些 Boss，分别用什么拾取专精
- 若可达性不足，哪些活动本周不该勉强投入

### 6. 未来 1-3 周路线
- 未来 1-3 周的主 roll 池
- 什么情况下继续追同一个池
- 什么情况下转向第二主池
- 如果本周拿到关键武器 / 饰品，后续优先级如何变化

### 7. Roll 优先级
按 `S / A / B / 不建议` 分层，每项都写：
- 活动
- 目标物品
- 推荐拾取专精
- 目标池是否因拾取专精变小
- 为什么值这个优先级
- 适合哪类玩家

### 8. 宝库代币建议
- 这周应该立刻拿哪件宝库，还是拿 6 代币
- 是否应该先 roll 再领宝库；如果必须先拿 6 代币，则明确写出原因
- 什么情况下 6 代币换额外晦暗虚空核心
- 什么情况下直接拿宝库装备或其他奖励更划算
- 是否建议先 `/simc`

### 9. 长期毕业、现实毕业与配装参考
- 快速接近全身 BIS 时，优先补哪些位
- 哪些理论 BIS 当前应降为远期目标，哪些更适合按现实毕业处理
- 制作装建议是 `现在做`、`等本周关键 roll / 宝库后再做`、还是 `暂时不做`
- 上述制作建议是基于 `当前战力优先` 还是 `长期保值优先`
- 制作装 / 催化剂 / off-piece 会如何改变路线
- 高端样本 / 攻略的属性分配与当前角色的差距
- 高端样本 / 攻略的常见配装与当前角色的差距

### 10. 备选路线
- 如果打不到某个 Boss / 某个层数，用什么次优副本替代
- 如果套装进度提前完成，优先级如何变化
- 如果改用另一套 Hero talents，优先级是否变化
- 如果是小号 / 低装等角色，路线如何保守化

## 数据源优先级

每次都联网核实，不要凭记忆回答当前版本。

分三类取材，不要把角色当前状态排到通用 BIS 后面：

1. `版本与规则核实`
- 用户指定来源
- Wowhead 的 Voidforge / Bonus Roll 规则页
- Wowhead 的 dungeon / raid rewards guides
- 官方游戏内冒险指南 / 地城手册的拾取专精掉落信息，若用户提供截图则优先使用

2. `专精与主流样本`
- Wowhead 对应专精页、`Talent Builds`、`Overview`、`Hero Talents`
- Mythicstats 或同类高分玩家装备聚合站
- Bloodmallet 的 trinkets / secondary distributions / consumables 等模拟页，但只作为补充理论源
- Warcraft Logs 团本日志、Boss 排名、Boss 维度构筑
- Warcraft Logs Mythic+ 排名、Statistics、Talents、Trinkets 维度

3. `角色当前周决策`
- 用户提供的 `/simc` 导出、角色面板、宝库截图或装备列表
- 用户提供的 `官方英雄榜 / 角色页 URL`，用于读取当前穿着装备、装等、专精、进度与可见成就；若和用户手动补充冲突，优先采用用户更新更近的说明
- 当前可达性、已做制作装、套装进度、已通过晦暗虚空核心拿到过的同池物品
- 当前拾取专精、可切换拾取专精、每个拾取专精在目标活动里的可 roll 池大小
- 做最终周决策时，角色当前状态的权重高于通用 BIS 长表；通用 BIS 只负责判断候选目标是否值得投入 roll 币
- 如果问题聚焦 `饰品 / 副属性分布`，可以用 Bloodmallet 作为补充模拟源；但它不能单独决定长期 R 币路线，也不能替代 `/simc`

查询某个专精在某个 Boss / 大秘境的可 roll 池时，优先按这个顺序：
- 先查目标物品页面，确认来源活动、装备类型、是否有职业 / 专精限制
- 再查副本 / Boss 掉落页或 Wowhead 过滤结果，按职业与拾取专精筛掉不适合该专精的物品
- 再用游戏内冒险指南或用户截图复核目标是否在该拾取专精池内
- 如果有可靠 addon / 工具能展示 Voidforge 剩余池，可以作为辅助，但最终要说明它不是官方来源

如果问题是 `饰品二选一 / 饰品优先级 / 理论副属性分布`，额外参考这个顺序：
- 先看 Wowhead 对该专精、该玩法下的饰品结论
- 再看 Bloodmallet 的对应 spec / fight style 模拟页，确认理论单体或 Casting Patchwerk 倾向
- 再用 Mythicstats / WCL 验证这些饰品是否真的在当前 M+ 或团本样本中高频出现
- 若 Bloodmallet 与实战样本冲突，以 `玩家内容定位 + 可达性 + 实战样本 + /simc` 综合裁决，不要只看纯 Patchwerk 模拟

若职业页之间互相冲突：
- 先比更新时间，较新的职业页优先
- 若较新的 `Overview` / `Talent Builds` 与较旧的 `Hero Talents` 冲突，优先采用较新的结论，并在答案里说明“旧页尚未同步”
- 再用 Mythicstats / 日志聚合验证主流 Hero talents 与装备选择
