# 输入完整度规则

## 一、基础原则

- `/simc` 不是强制前置条件。
- 没有 `/simc` 时，仍然可以根据版本规则、主流 BIS、玩家可达性给出高质量方向性建议。
- 但当问题进入“精细比较”时，应该主动提醒用户：`如果要更精确，最好提供 /simc`。

## 二、三档输入

### 基础档

用户通常只给：

- 职业专精
- Hero talents
- 主玩法
- 大致装等
- 可打内容

这时可以回答：

- 该优先 roll 哪个副本 / Boss
- 该优先追饰品、武器还是 off-piece
- 宝库大方向更偏拿装备还是拿代币

这时不要回答得过于精确：

- 不要硬判两件接近的绿字散件谁一定更强
- 不要把“方向性建议”包装成“绝对数值最优”

### 进阶档

用户再补：

- 当前双饰品
- 当前武器 / 副手
- 当前套装进度
- 当前最弱槽位
- 当前宝库候选
- 当前制作装状态

这时可以回答：

- 更稳的周计划
- 宝库拿装备还是拿代币
- 哪个 roll 目标更适合本周
- 制作装是否会挤掉某个 roll 槽位

### 精确档

用户提供：

- `/simc` 导出
- 宝库截图
- 当前待选装备

这时可以回答：

- 两件武器 / 饰品 / 戒指 / 项链谁更优
- 宝库装备与额外核心的精细取舍
- Hero talents 切换下的细微差别
- 制作装与掉落装的精细比较

## 三、什么时候必须建议 `/simc`

以下情况，不建议只靠肉眼：

- 两件高等级武器二选一
- 两件饰品二选一
- 多件戒指 / 项链 / 披风 / 护腕这类纯绿字散件比较
- 宝库装备 vs 6 代币换额外核心
- Hero talents 切换导致属性权重可能变化
- 制作装会直接改变套装位 / off-piece 规划

## 四、建议用户怎么补信息

如果用户没有 `/simc`，至少建议补这些：

- 当前双饰品
- 当前主手 / 副手
- 当前 2 件套 / 4 件套进度
- 当前宝库候选
- 当前已做制作装

可以直接这样说：

`如果你暂时没有 /simc，我也可以先按你当前双饰品、武器、套装进度和宝库候选给你做较稳的判断。`

## 五、输出措辞规范

### 没有 `/simc`

用词应偏保守：

- `更建议`
- `大概率更值`
- `按当前主流样本更偏向`
- `方向上优先`

避免：

- `严格最优`
- `必然提升最大`
- `绝对 BIS`

### 有 `/simc`

可以更明确：

- `当前角色下更优`
- `更接近精确结论`
- `这一步可以按 sim 结果执行`
