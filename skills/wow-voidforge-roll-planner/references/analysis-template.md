# 输出骨架

按下面结构组织答案：

## 零、玩家画像与假设
- 主号 / 小号
- 团本玩家 / M+ 玩家 / 双修玩家
- 输入完整度：基础档 / 进阶档 / 精确档
- 当前装等
- 当前 2 件套 / 4 件套进度
- 当前可稳定完成的 M+ 层数
- 当前可稳定击杀的团本难度 / Boss 段位
- 当前宝库候选
- 当前制作装状态
- 若信息不足，明确写出默认假设

## 一、版本与规则确认
- 补丁与赛季：明确写出绝对版本号与日期
- 晦暗虚空核心：每周来源、每周上限、单次消耗
- 宏伟宝库：6 代币是否能换额外 roll，是否占每周上限

## 二、该专精的装备逻辑
- 属性优先级：只写会影响 roll 决策的内容
- Hero talents：分别写团本 / M+ 主流选择
- Hero talents 是否改变装备池：明确写“基本不变”或“需要拆分两套”
- 数据源加权：说明这一版结论主要参考 Wowhead、Mythicstats、WCL 团本、WCL Mythic+ 中的哪两类
- 4 件套默认放在哪些槽位
- 哪些槽位通常不建议用 roll 币追
- 制作装是否会直接挤占某个掉落位

## 三、目标物品分级
先按下面三类整理，再做总优先级：

- 团本与 M+ 都强
- 更偏团本
- 更偏 M+

### S 级
- 目标物品
- 来源活动
- 推荐理由
- 适合玩家层级

### A 级
- 同上

### B 级
- 同上

### 不建议主动 roll
- 套装可转化位
- 可制造位
- 只提供普通绿字升级的散件

## 四、宝库与代币决策
- 这周宝库有没有应该直接拿的选项
- 若没有，6 代币换额外核心是否更值
- 是否建议先 `/simc`
- 如果没有 `/simc`，还缺哪些最关键装备信息

## 五、实战路线
- 先刷 / 先 roll 什么
- 第一优先拿到后，下一步转向什么
- 如果没有合适宝库装备，6 代币是否建议换额外 roll
- 如果切换到另一套 Hero talents，哪些结论会变，哪些不会变
- 如果是小号 / 低装等号，路线如何保守化

## 六、假设与限制
- 玩家是否能打英雄团 / 史诗团
- 玩家是否能稳定打到需要的 M+ 层数
- 哪些结论依赖当前主流样本而非硬性机制
- 中文名是否已核到 Wowhead 中文页；若没有，第一次提及时用“中文描述（英文原名）”
- 哪些分歧需要 `/simc` 才能严谨判断
- 当前结论属于方向性建议，还是接近精确比较
